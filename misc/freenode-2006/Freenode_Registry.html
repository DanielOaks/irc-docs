Summary

freenode registry is intended as a web-based, modular framework to replace dancer-services, aka. theia. Broad functions to be provided include:

  • User and group registration services and an authentication framework extendable in a modular way.
  • Incident tracking service with real time notification on and off the freenode network, tightly-coupled to volunteer scheduling services.
  • 24/7 volunteer scheduling services.
  • News and documentation pages, with RSS support for update notification.
  • IRC nick and channel reservation and configuration.
  • XML-RPC and/or SOAP support for bot access; FOAF support for entity description.

NOTE: This web interface in no way replaces the chat component of freenode

[edit]

Call for Assistance

PDPC and freenode are looking for coders with time availability to work on freenode registry. Familiarity with PostgreSQL and/or MySQL, web application systems, sockets, RSS, XML-RPC, SOAP and XML (including FOAF) are all useful. In addition to coding help, we'll need assistance with web design and graphics. Please be willing to sign a copyright assignment similar in character to those required by the FSF. See below for discussion of language requirements.

Language Requirements

The initial intent was to code the initial implementation in Python. FreenodeRegistryLanguage details the reasoning involved. But progress has been slow. What we really need are coders with considerable time availability and an interest in firming up requirements and producing code. Work on the Python version is ongoing. But if you would like to short circuit the process, contact lilo and let him know your thoughts on the matter. Be prepared to produce a fair amount of code, though, regardless of the language.
