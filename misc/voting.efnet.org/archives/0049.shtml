services.int reorganization/replacement of chanfix

This vote will ultimately lead to the creation of one
centralized services, and one set of backup services.
Openchanfix has proven itself stable, has been ported to
ratbox2, and by the time this vote comes to a
conclusion be ready to go into full production mode.

If this vote passes, open chanfix (OCF) will replace the
current code running on services.int. Garion (as a
representative from the coding team) plus a representative
nominated and elected by each region will form the new
group of services admins responsible for maintaining and
operating the new services.int. These elected
representatives, and Garion as representative of the
coding team, will then assume full responsibility for
maintaining all services related matters. This includes
replacing JUPES with an actually operational
OpenJupes code (when ported to rb2) and any future
developments requested by the admin community as a whole.

A backup set of services will be run from the current
services.eu location, but be renamed backup.services.int.
It will run in passive mode incase of extended failure by
services.int.
