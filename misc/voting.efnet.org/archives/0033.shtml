Date of conclusion : 30 Jun 2003
Result: PASSED (yes: 16, no: 11, abstain: 4)
Description:


The following vote has closed:
                                                                                
Operspy accountability
                                                                                
While clients on a local server should rightfully be
policed by that server staff alone, Information obtained
by use of operspy features is global. Channels being
global in nature might reveal information sensitive to
users on another server, outside jurisdiction of the
initiating party. This vote calls for accountibility on
both local and global uses of operspy features.
                                                                                
Vote YES will require all uses of operspy, local and
global, to be broadcast and logged globally.
                                                                                
Vote NO will require only global uses of operspy to be
broadcast and logged.
