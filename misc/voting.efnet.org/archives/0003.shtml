Topic: Voting admins with multiple servers
Called by: Xref (irc.umich.edu)

Called on: 01/30/01
Ended on: 02/13/01
Status: PASSED (yes: 16, no: 1, abstain: 0, elig: ~25)

Description:


Each organization that runs an EFNet server is given 1 vote, provided that the
person voting does not already vote for another server.

Specific cases, including but not limited to admins appointing "voting admins" to
vote on their behalf for multiple servers will be dealt with on a case by case
basis as detailed in the previous vote:

4) In case one admin represents multiple servers that can be
seen as one, we (the admins) should discuss the number of
votes that server gets. If there is failure to agree, it will be decided by vote.
