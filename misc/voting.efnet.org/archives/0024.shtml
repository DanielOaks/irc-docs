Description:

Maximum 3 Chanfix Opers per server entity.

Allow 3 Chanfix Opers at maximum per server entity,
including the admin or admins.  When an admin picks an Oper
to be a Chanfix Oper (as opposed to picking him/herself), a
notice must be sent 72 hours in advance to a global Admins
list (voting?) to allow for objections.

This idea is based off the way EU allows for objections
against any new global opers.  It won't be possible to end
up with 5 ppl per server with access.  This should make
it not necessary to make an oper a co-admin just for the
sake of chanfix access, (or access to the mailing lists for
that matter), etc.
    As some would say 'If you don't have conf access, you're
not an admin'...
